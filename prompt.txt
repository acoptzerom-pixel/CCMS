# Product Requirements Document (PRD): Consulting Center Management System (CCMS)  (Updated)

## 1. ข้อมูลโครงการ (Project Overview)
**ชื่อโครงการ:** Consulting Center Management System (CCMS)
**เป้าหมาย:** ระบบบริหารจัดการศูนย์ให้คำปรึกษาฯ ของศาลเยาวชนและครอบครัว แบบ Real-time โดยเน้นความง่ายและสะดวกในการใช้งาน สามารถใช้ได้ทุกที่ทุกเวลา ได้ทุกอุปกรณ์ ผ่าน apps scripts ใน Google sheets
**เชื่อมต่อกับฐานข้อมูล Google sheets จากลิงค์ : https://docs.google.com/spreadsheets/d/1-D1Dpin3KTuH9oV1-cUyEDXuJ15gKaHi6z6K6MIYMBU/edit?usp=sharing

## 2. ระบบสมาชิกและการเข้าถึง (User Authentication)
* **Method:** ระบบสมัครและเข้าสู่ระบบด้วย Email และ Password
* **Security:** * รหัสผ่านต้องมีการเข้ารหัส (Hashing) ก่อนเก็บลงฐานข้อมูล
    * มีระบบจดจำการเข้ารหัส (Session/Token) เพื่อความสะดวกในการใช้งานผ่าน Browser


## 3. การจัดการข้อมูล (Data Integration)
* **Primary Source:** `https://docs.google.com/spreadsheets/d/1-D1Dpin3KTuH9oV1-cUyEDXuJ15gKaHi6z6K6MIYMBU/edit?usp=sharing`
* **Data Handling:**
    * หากมีการคำนวณ ให้แปลงค่าจาก String เป็น Decimal/Float ก่อนนำไปคำนวณ
* **Error Handling (กรณีแหล่งข้อมูลขัดข้อง):** * หากไม่สามารถดึงข้อมูลได้ ให้ระบบแสดง Notification หรือ Banner แจ้งเตือนว่า **"ขณะนี้แหล่งข้อมูลขัดข้อง โปรดตรวจสอบใหม่อีกครั้งในภายหลัง"**
    * ในระหว่างที่ข้อมูลขัดข้อง ให้แสดงข้อมูลล่าสุดเท่าที่มีอยู่ใน Cache/Database (ถ้ามี) พร้อมระบุเวลาที่อัปเดตล่าสุด

## 4. ระบบการจัดการข้อมูลแต่ละบุคคล (person Logic)
* **แบ่งสถานะการเข้าถึงฟังก์ชั่นแต่ละระบบตาม role โดย admin สามารถเข้าถึงได้ทุกฟังก์ชั่น 
* **มีฟังก์ชั่นในการเพิ่มข้อมูลต่าง ๆ และกำหนดสถานะของ user โดยมีเฉพาะ admin ที่สามารถเข้าถึงได้
* **มีฟังก์ชั่นเพิ่ม แก้ไข เปลี่ยนแปลง ลบ ข้อมูลของเด็กและเยาวชน โดยเริ่มจากการค้นหาข้อมูลจากเลขบัตรประชาชนก่อน (citizen_id) หากพบให้ดึงข้อมูลเดิมมา และแสดงข้อมูลคดีที่เชื่อมโยงกันให้เป็นสัดส่วนชัดเจน
* **มีฟังก์ชั่นเพิ่ม แก้ไข เปลี่ยนแปลง ลบ ข้อมูลคดีในแต่ละมาตรา และเชื่อมโยงกับข้อมูลเด็กและเยาวชนผ่านเลขบัตรประชาชนก่อน (citizen_id)
* **เพิ่มข้อมูลผลการตรวจวินิจฉัยทางจิตวิทยาเบื้องต้นเข้าไปด้วย

## 5. ขอบเขตงาน (Scope)

### ✅ สิ่งที่ควรทำ (In-Scope)
* **Responsive UI:** แสดงผลสวยงาม ทันสมัย และมีความน่าเชื่อถือ ทั้ง Mobile และ Desktop
* **Dashboard:** * แสดงข้อมูลภาพรวมโดยสามารถเลือกช่วงวันที่ที่ต้องการให้ข้อมูลแสดงได้ (ค่าเริ่มต้นให้เป็นวันที่ 1 มกราคาของปีปัจจุบัน จนถึงวันที่ปัจจุบัน)
    * แสดงกราฟวงกลมประเภทคดีที่มากที่สุด10อันดับแรก (ข้อมูลจากคอลัมน์ charge_no เชื่อมโยงกับ charge_statistics)
    * แสดงจำนวนเด็กและเยาวชนที่รับเข้ามาใหม่ โดยแยกเป็น เพศ (gender) อายุ (age_at_offense) อำเภอ (district_address)
    * แสดงจำนวนคดีที่เข้ามาใหม่ในแต่ละมาตราตาม black_date (มาตรา 73, มาตรา 90, มาตรา 132 วรรหนึ่ง, มาตรา 132 วรรคสอง, และมาตราอื่นๆหรือคดีจากศาลอื่น ตาม case_type_id เชื่อมโยงกับ case_type, sub_case_type เชื่อมโยงกับ sub_case_type_id)
    * แสดงจำนวนคดีที่ยุติในแต่ละมาตราตาม rad_date (มาตรา 73, มาตรา 90, มาตรา 132 วรรหนึ่ง, มาตรา 132 วรรคสอง, และมาตราอื่นๆหรือคดีจากศาลอื่น ตาม case_type_id เชื่อมโยงกับ case_type, sub_case_type เชื่อมโยงกับ sub_case_type_id)
    * แสดงจำนวนการกระทำผิดซ้ำจาก recidivism


### ❌ สิ่งที่ไม่ควรทำ (Out-of-Scope)
* ไม่มีการเชื่อมต่อ API กับฐานข้อมูล
* ไม่มีระบบ Backup API (หาก source หลักล่ม ให้แจ้งเตือนตามที่ระบุในข้อ 3)

## 6. แนวทางการออกแบบ (UI/UX Guidelines)
* **Less is More:** ตัวอักษรน้อย เน้นตัวเลขและสัญลักษณ์ที่เข้าใจง่าย


## 7. ข้อเสนอแนะเรื่องความปลอดภัย (Initial Security)
 **Access Control:** ผู้ใช้แต่ละคนต้องมองเห็นเฉพาะข้อมูลเด็กและเยาวชน/คดี ที่ตนเองเป็น counselor เท่านั้น (ตาม user_id ในตาราง tb_cases)
 **Secure API:** หากมีการทำ Backend API สำหรับดึงข้อมูล ต้องมีระบบป้องกันการเรียกข้อมูลจาก Unauthorized User

## 8. ทำให้สามารถเปิดระบบทางออนไลน์ได้ จากลิงค์ของ google script สามารถเปิดไอ้ในทุกอุปกรณ์ ทุกระบบปฏิบัติการ ทั้ง window android และ ios
