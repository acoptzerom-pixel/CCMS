# สรุปการดำเนินงาน (Walkthrough) - แยกที่อยู่ภูมิลำเนาและเพิ่มฟิลด์ข้อมูลเยาวชน/ผู้ปกครอง

## ปัญหาและหัวข้อการปรับปรุง

### 1. การปรับปรุงช่องกรอกที่อยู่ภูมิลำเนาแบบแยกหัวข้อ
* **ความต้องการ**: ปรับปรุงจากเดิมที่เป็นช่องกรอกแบบข้อความยาวช่องเดียว (Textarea) ให้เป็นแบบแยกกรอกข้อมูลหลายช่องเพื่อการบันทึกฐานข้อมูลที่สมบูรณ์และถูกต้องตามภูมิศาสตร์ สำหรับวิเคราะห์สถิติตามอำเภอ
* **การดำเนินการ**:
  - **ปรับโครงสร้างฐานข้อมูลตาราง `tb_defendants`**: ปรับปรุงฟังก์ชัน `upsertDefendant` ใน [Code.gs](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Code.gs#L1556) โดยเพิ่มกลไก `ensureColumnsExist` ตรวจสอบและสร้างคอลัมน์เก็บข้อมูลแยกส่วน ได้แก่ `number_address` (บ้านเลขที่), `village_address` (หมู่บ้าน), `village_number_address` (หมู่ที่), `road_address` (ถนน), `sub_district_address` (ตำบล/แขวง), `district_address` (อำเภอ/เขต), `province_address` (จังหวัด), และ `postcode_address` (รหัสไปรษณีย์) โดยอัตโนมัติ
  - **ปรับอินเตอร์เฟซการกรอกข้อมูล**:
    - **หน้าแก้ไขประวัติเยาวชน (Edit Modal)** ใน [Index.html](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Index.html#L1410): ลบช่องกรอกที่อยู่แบบ Textarea เดิมออก เปลี่ยนเป็นแบบฟอร์มตารางกริดที่แบ่งช่องกรอกแยกเป็น 8 ช่องย่อยอย่างสวยงามและเป็นสัดส่วน
    - **แบบฟอร์มเพิ่มเด็กและเยาวชนใหม่ลงระบบเมื่อค้นหาไม่พบ (Register Form)** ใน [Index.html](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Index.html#L820): เปลี่ยนเป็นแบบแยกกรอกข้อมูล 8 ช่องเช่นเดียวกัน
  - **ปรับปรุงสคริปต์หน้าบ้าน**: เพิ่มตัวแปร Reactive ย่อยใน `defForm` ใน [Scripts.html](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Scripts.html#L440) เพื่อให้รองรับการผูกข้อมูล (Data Binding) สำหรับส่งค่าผ่าน API
  - **จัดรูปแบบสำหรับแสดงผลที่อยู่ปัจจุบัน (แก้ไขใน V117)**: 
    - ปรับปรุงฟังก์ชัน `formatDefendantAddress` ใน [Code.gs](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Code.gs#L1438) เพื่อนำฟิลด์ย่อยเหล่านี้กลับมารวมเข้าด้วยกันเป็นประโยคที่ถูกต้องตามรูปแบบที่อยู่อย่างเหมาะสม (พร้อมระบบช่วยกรอกคำนำหน้า ต., อ., จ. หรือ เขต/แขวง ให้โดยอัตโนมัติเมื่อค่าที่กรอกมาไม่มีตัวย่ออยู่แล้ว และรองรับกรุงเทพฯ)
    - ปรับปรุงฟังก์ชัน `searchDefendantByCitizenId` และ `getAllDefendantsWithCases` ใน [Code.gs](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Code.gs) ให้เรียกใช้งาน `formatDefendantAddress` เสมอ เพื่อให้การโหลดข้อมูลและค้นหาประวัติได้ที่อยู่ปัจจุบันที่จัดรูปแบบเต็มเรียบร้อย โดยไม่ดึงข้อมูลดิบและป้องกันที่อยู่ขึ้นว่า "ไม่ระบุที่อยู่" บนหน้าการ์ดข้อมูล

### 2. เพิ่มข้อมูลประวัติส่วนตัวเยาวชนและผู้ปกครองเพิ่มเติม (รวมความสัมพันธ์ใน V117)
* **ความต้องการ**: แสดงและแก้ไขฟิลด์: ชื่อผู้ปกครอง (`guardian_name`), ความสัมพันธ์ (`relationship`), เบอร์โทรศัพท์เยาวชน (`phone_number`), เบอร์โทรศัพท์ผู้ปกครอง (`phone_number_guardian`), อาชีพเยาวชน (`occupation`), และอาชีพผู้ปกครอง (`occupation_guardian`)
* **การดำเนินการ**:
  - **หน้าแสดงข้อมูลรายละเอียดเด็ก (Detail Card)** ใน [Index.html](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Index.html#L727): เพิ่มการแสดงผลข้อมูลทั้ง 6 ฟิลด์นี้ลงในตารางข้อมูลทั่วไป โดยสำหรับความสัมพันธ์ (`relationship`) ได้ดึงมาจัดรูปแบบและแสดงต่อท้ายชื่อผู้ปกครองเป็น `ชื่อผู้ปกครอง (ความสัมพันธ์)` เช่น `นางสาวปรียา ดีใจ (มารดา)`
  - **การกรอกและแก้ไขข้อมูล**: เพิ่มอินพุตฟิลด์ "เกี่ยวข้องเป็น" (`relationship`) คู่กับชื่อผู้ปกครอง ลงในทั้ง **แบบฟอร์มเพิ่มเยาวชนรายใหม่** และ **หน้าต่างแก้ไขประวัติ (Edit Modal)** ใน [Index.html](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Index.html) ถัดจากช่องชื่อผู้ปกครอง
  - **ตัวเก็บข้อมูลฝั่งเซิร์ฟเวอร์**: มั่นใจว่าฟิลด์เหล่านี้รวมถึง `relationship` ซิงค์และเขียนลงตารางชีท `tb_defendants` อย่างครบถ้วน

### 3. ปรับเปลี่ยนป้ายข้อความประวัติ และเปิดให้แก้ไขและบันทึกประวัติเพิ่มเติม (แก้ไขใน V118)
* **ความต้องการ**: 
  - เปลี่ยนชื่อป้ายข้อความจาก "ประวัติการรักษาแพทย์" เป็น **"ประวัติการรักษา"** (เชื่อมโยงกับ `medical_history`)
  - เปลี่ยนชื่อป้ายข้อความจาก "ประวัติการยุ่งเกี่ยวสารเสพติด" เป็น **"ประวัติการใช้สารเสพติด"** (เชื่อมโยงกับ `drug_history`)
  - เพิ่มการแสดงผลและการแก้ไขข้อมูล **"ความสนใจ/ความสามารถพิเศษ"** (เชื่อมโยงกับ `interested`)
  - เพิ่มช่องอินพุตข้อมูลทั้ง 3 รายการนี้ในส่วนการลงทะเบียนประวัติและหน้าต่างแก้ไขประวัติเยาวชน
* **การดำเนินการ**:
  - **การปรับเปลี่ยนใน Index.html**:
    - อัปเดตป้ายแสดงผลในประวัติเด็กหลักและหน้ารายละเอียดคัดกรองจากหน้าต่างปฏิทิน (`calendarDetailDef`)
    - เพิ่มช่อง Textarea 3 ช่องใหม่สำหรับประวัติการรักษา, ประวัติการใช้สารเสพติด, และความสนใจ/ความสามารถพิเศษ ในทั้งแบบฟอร์มเพิ่มประวัติใหม่และหน้าต่างแก้ไขประวัติเยาวชน
  - **ตัวตรวจสอบคอลัมน์ฝั่งเซิร์ฟเวอร์ (Code.gs)**: เพิ่ม `medical_history`, `drug_history`, และ `interested` ลงในรายการตรวจสอบ `ensureColumnsExist` เพื่อรองรับการสร้างหัวตารางในสเปรดชีตอย่างปลอดภัยเมื่อมีการลงทะเบียนเด็กใหม่

### 4. เพิ่มข้อมูลประวัติการศึกษา ปรับคำอธิบายที่อยู่ จัดเรียงข้อมูลเด็กไว้บน ผู้ปกครองไว้ล่าง (แก้ไขใน V119)
* **ความต้องการ**:
  - เพิ่มและเปิดให้แก้ไขข้อมูล **"ระดับการศึกษา"** (`education_level`) และ **"โรงเรียน/วิทยาลัย"** (`school`)
  - จัดเรียงลำดับหัวข้อการแสดงผลและการกรอกข้อมูลใหม่ โดยข้อมูลทุกอย่างของเยาวชนจะอยู่ **ส่วนบนสุด** ทั้งหมด และข้อมูลของผู้ปกครองย้ายไปแสดงที่ **ล่างสุด**
  - ในหน้าแก้ไขข้อมูล ปรับป้ายกำกับส่วนที่อยู่แยกช่องจากเดิม "ที่อยู่ภูมิลำเนา (เพื่อประมวลผลคำนวณสถิติอำเภอ)" เป็น **"ที่อยู่ปัจจุบันของเด็ก/เยาวชน"**
* **การดำเนินการ**:
  - **การจัดเรียงและการแสดงผล**: ปรับการเรนเดอร์ใน [Index.html](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Index.html) (หน้ารายละเอียดหลัก, ฟอร์มลงทะเบียนใหม่, หน้าต่างแก้ไขประวัติ, และหน้าปฏิทินนัดหมาย) เพื่อเรียงลำดับฟิลด์ของเยาวชนไว้ข้างบน (มีระดับการศึกษาและโรงเรียนเพิ่มเข้ามา) และนำส่วนการกรอก/แสดงผล ข้อมูลผู้ปกครอง (ชื่อผู้ปกครอง, ความเกี่ยวข้อง, เบอร์โทรศัพท์, อาชีพผู้ปกครอง) ย้ายไปอยู่ในกล่องการ์ดด้านล่างสุดของฟอร์ม
  - **การอัปเดตเซิร์ฟเวอร์และโมเดล**:
    - เพิ่มการตรวจสอบคอลัมน์ `education_level` และ `school` ในฟังก์ชัน `ensureColumnsExist` ของ [Code.gs](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Code.gs)
    - ผูกค่า `defForm.school` และ `defForm.education_level` เพื่อส่งไปบันทึกฝั่งเซิร์ฟเวอร์ใน [Scripts.html](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Scripts.html)

### 5. ปรับเปลี่ยนหัวข้อผลประเมินทางจิตวิทยาเบื้องต้น เป็น การประเมินความเสี่ยงในการกระทำผิดซ้ำ (แก้ไขใน V120)
* **ความต้องการ**:
  - เปลี่ยนชื่อป้ายข้อความจาก "ผลการตรวจวินิจฉัยทางจิตวิทยาเบื้องต้น" เป็น **"การประเมินความเสี่ยงในการกระทำผิดซ้ำ"** (เชื่อมโยงกับ `risk_assessment`)
  - ปรับรูปภาพสัญลักษณ์ (Icon) ข้างหน้าข้อความให้สื่อถึงการประเมินความเสี่ยงเชิงปริมาณ (ใช้ไอคอนมาตรวัดความเร็ว/ความเสี่ยง `fa-gauge-high`)
  - ในหน้าบันทึกและแก้ไขข้อมูล ปรับเปลี่ยนจากช่องใส่ข้อความฟรีสไตล์ (Textarea) เป็นช่อง Dropdown รายการตัวเลือกแบบสวยงาม
* **การดำเนินการ**:
  - **การแสดงผลและการ์ดสรุปประวัติ**: อัปเดตใน [Index.html](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Index.html) ให้แสดงแถบสีข้อความ (Badge/Tag) ตามระดับความเสี่ยง of เยาวชนที่ได้รับบันทึกโดยอัตโนมัติ (เช่น ความเสี่ยงสูงจะใช้สีแดงอ่อน, ความเสี่ยงต่ำใช้สีเขียวอ่อน)
  - **การอัปเดตสิทธิ์และตาราง**:
    - เพิ่มการยืนยันคอลัมน์ `risk_assessment` ในตาราง `tb_defendants` ผ่านฟังก์ชัน `ensureColumnsExist` ของ [Code.gs](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Code.gs)
    - ผูกตัวเลือกในกล่อง Dropdown หน้าแก้ไขและหน้าลงทะเบียนเด็กเข้ากับพร็อพเพอร์ตี้ `defForm.risk_assessment` ใน [Scripts.html](file:///c:/Users/LENOVO/OneDrive/Cz/โปรแกรมศูนย์ให้คำปรึกษา/Scripts.html) พร้อมกำหนดค่าดีฟอลต์เป็นระดับความเสี่ยงต่ำ

### 6. แยกการ์ดข้อมูลผู้ปกครอง และอัปเดตป้ายข้อความแสดงผลและกรอกข้อมูล (แก้ไขใน V121)
* **ความต้องการ**:
  - ในหน้าแสดงผลประวัติเด็ก แยกหัวข้อ "ชื่อผู้ปกครอง", "เบอร์โทรศัพท์ผู้ปกครอง", และ "อาชีพผู้ปกครอง" ออกมาเป็น **อีกการ์ดหนึ่งแยกต่างหาก** อย่างเป็นสัดส่วนชัดเจน
  - เปลี่ยนชื่อป้ายกำกับฟิลด์ (ทั้งในตารางแสดงผลและแบบฟอร์มกรอกข้อมูล):
    - `ชื่อ-สกุลจริง:` เปลี่ยนเป็น **`ชื่อ-สกุล (ชื่อเล่น/ฉายา):`**
    - `เบอร์โทรศัพท์เยาวชน:` เปลี่ยนเป็น **`เบอร์โทรศัพท์:`**
    - `อาชีพเยาวชน:` เปลี่ยนเป็น **`อาชีพ (ตำแหน่ง):`**
    - `อาชีพผู้ปกครอง:` เปลี่ยนเป็น **`อาชีพผู้ปกครอง (ตำแหน่ง):`**
* **การดำเนินการ**:
  - **ปรับโครงสร้างอินเตอร์เฟซ (Index.html)**:
    - ปรับโครงสร้างฝั่งซ้ายของหน้ารายละเอียดเด็กโดยสร้าง Layout Stacking ครอบการ์ด 2 ใบ: การ์ดแรกแสดงเฉพาะรายละเอียดเยาวชนล้วน ๆ และเพิ่มการ์ดใบที่สองแยกแสดงรายละเอียดข้อมูลผู้ปกครอง (ใช้สัญลักษณ์ `fa-people-roof`)
    - อัปเดตคำแสดงผลในฟอร์มกรอกประวัติใหม่และหน้าแก้ไขข้อมูลผู้มีสิทธิ์ให้ตรงตามป้ายข้อความที่ระบุใหม่เรียบร้อยแล้ว

### 7. เพิ่มการ์ดความเห็นเพิ่มเติมของผู้ให้คำปรึกษาพร้อมระบบบันทึกทันที (แก้ไขใน V122)
* **ความต้องการ**:
  - เพิ่มกล่องการ์ด **"ความเห็นเพิ่มเติมของผู้ให้คำปรึกษาที่มีต่อเด็กและเยาวชน"** นำไปวางไว้ใต้การ์ดข้อมูลผู้ปกครอง
  - สามารถป้อนข้อมูลความคิดเห็นและคลิกบันทึกได้ทันทีจากหน้าจอแสดงผลประวัติโดยไม่ต้องเข้าหน้าแก้ไขข้อมูลหลัก
  - จำกัดสิทธิ์การแก้ไข/บันทึกความเห็นเฉพาะผู้ใช้บทบาท: **ผู้ให้คำปรึกษา (counselor)**, **นักจิตวิทยา (psychologist)**, และ**แอดมิน (admin)** เท่านั้น
* **การดำเนินการ**:
  - **การปรับฐานข้อมูล (Code.gs)**: เพิ่มการตรวจสร้างคอลัมน์ `counselor_comments` อัตโนมัติ และสร้างฟังก์ชันความปลอดภัย `updateCounselorComments` บนเซิร์ฟเวอร์เพื่อเช็คเซสชันและบทบาทผู้เขียน
  - **การ์ดและปุ่มบันทึกด่วน (Index.html)**: นำการ์ดใบที่สามมาซ้อนต่อท้ายฝั่งซ้าย โดยจะประเมินสิทธิ์ของผู้ใช้ปัจจุบัน
  - **การผูกฟังก์ชัน JavaScript (Scripts.html)**: ผูกฟังก์ชัน `saveCounselorComments` เข้ากับ API ของระบบพร้อมแสดง SweetAlert2 เมื่อบันทึกสำเร็จโดยไม่มีการรีโหลดหน้าเว็บ

### 8. ปรับปรุงความสม่ำเสมอของสีการ์ดความเห็นเพิ่มเติม และเชื่อมโยงข้อมูลกับคอลัมน์ `counselor_note` (แก้ไขใน V123-128)
* **ความต้องการ**:
  - ปรับปรุงสีและการออกแบบของการ์ด "ความเห็นเพิ่มเติมของผู้ให้คำปรึกษาฯ" เพื่อให้กลมกลืนและเป็นหนึ่งเดียวกับการ์ดประวัติเยาวชนและการ์ดข้อมูลผู้ปกครองอื่น ๆ (ไม่มีการฝืนสีพื้นหลังหรือเส้นขอบ override)
  - เปลี่ยนการเชื่อมโยงข้อมูลไปที่คอลัมน์ **`counselor_note`** ในตาราง `tb_defendants` แทนคอลัมน์ `note` เดิม
  - อนุญาตให้ผู้มีสิทธิ์ (แอดมิน, นักจิตวิทยา, และผู้ให้คำปรึกษา) สามารถเขียนและบันทึกข้อมูลได้
  - สำหรับบทบาท **ผู้อำนวยการศูนย์ (director)** ให้สามารถ **อ่านข้อมูลได้อย่างเดียว** ไม่สามารถแก้ไขหรือบันทึกได้
  - **แก้ไขการทำงานของปุ่มบันทึกและการผูกฟังก์ชัน (แก้ไขใน V128)**: 
    - จัดสิทธิ์แบบฟอร์มและการผูกฟังก์ชันหน้าบ้าน ให้ใช้งานตัวแปร `tempCounselorNote` ร่วมกับฟังก์ชัน `startEditCounselorNote` และ `cancelEditCounselorNote` ตรงกันระหว่างไฟล์ `Index.html` และ `Scripts.html`
    - แก้ไขฟังก์ชันการยิง API `updateCounselorComments` ใน `Scripts.html` โดยส่งตัวแปรยืนยันตัวตน `session.value.token` ไปทางฝั่งหลังบ้านอย่างสมบูรณ์เพื่อป้องกัน Error การหมดอายุของสิทธิ์
    - ปรับปรุงกลไกตรวจสอบเลขบัตรประชาชนใน `Code.gs` ให้ใช้ฟังก์ชัน `cleanCitizenId` ทั้งสองฝั่งเทียบเคียงกันอย่างแม่นยำ ไม่ติดข้อจำกัดของเครื่องหมายขีดคั่นคดี
    - ทำแถบปุ่มแก้ไขและบันทึกความเห็น: แบบปกติแสดงปุ่มสีน้ำเงิน "แก้ไขความเห็น", แบบแก้ไขแสดงปุ่มแดง "ยกเลิก" และปุ่มเขียว "บันทึกความเห็น" ครบถ้วนตามโจทย์
